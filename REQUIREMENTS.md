# BSL-Station Requirements

> 版本：V1 正式方案
> 日期：2026-06-24

## 1. 项目目标

`BSL-Station` 是一个面向身边人使用的 Windows 原生游戏启动器。

V1 的目标不是重做一套全新的启动器壳，而是在现有 `Starward` 成熟基础上，做一个统一管理多款游戏的启动器，并尽量直接复用现成开源实现，避免重复造轮子。

## 2. 正式方案

- 正式主工程：`BSL-Station-Starward/src/Starward`
- 正式 UI 主参考：`refs/Starward`
- 当前首页与主交互继续基于 `Starward` 结构改造
- 米哈游游戏尽量直接复用 `Starward` 原生能力
- 第三方游戏 UI 尽量接到与米哈游游戏一致的逻辑层，而不是另起一套界面体系

不再继续的方向：

- 不再以 `XianYuLauncher` 作为正式主壳推进
- 不再维护另一套空白 WinUI 首页方案
- 不再并行维护多套 UI 主方案

## 3. 目标平台

- 平台：Windows
- 首发语言：简体中文
- 区服目标：国服优先，官服优先按国服路线落地

## 4. V1 游戏范围

V1 只做以下 6 个游戏：

- 原神
- 崩坏：星穹铁道
- 绝区零
- 鸣潮
- 明日方舟
- 明日方舟：终末地

V1 明确不做：

- `崩坏3`
- 其他未列出的游戏
- 其他/未适配游戏导入功能
- 国际服大范围适配

## 5. 技术路线

- 桌面技术：`.NET + WinUI 3`
- 根命名空间：继续沿用 `Starward`
- 架构目标：保留 `Starward` 现有成熟能力，增量接入 BSL 多游戏适配层
- 发布形态：安装版 + 便携版

## 7. V1 功能要求

### 7.1 首发 6 游戏共同目标

对这 6 个游戏，V1 尽量支持以下能力：

- 下载
- 更新
- 预下载
- 启动
- 修复
- 卸载
- 检测已安装并导入
- 手动导入已有目录
- 自定义安装目录
- 自定义启动程序
- 自定义启动参数

### 7.2 队列要求

- 同一时间只执行 1 个下载类任务
- 支持排队
- 支持失败重试
- 支持断点续传
- 启动器重开后尽量恢复下载类任务状态

### 7.3 空间检查

- 下载、更新、预下载前做磁盘空间检查
- 估算包含下载包、解压、补丁和缓存的额外空间

### 7.4 缓存策略

- 默认删除普通下载缓存
- 只有更新或预下载后处理失败时，才提示用户清理残留缓存
- 非米哈游游戏需要有明确的残留缓存提示和手动清理入口

## 8. 分游戏适配策略

### 8.1 原神 / 崩坏：星穹铁道 / 绝区零

- 目标：完整适配
- 实现策略：优先直接复用 `Starward` 原生下载、更新、预下载、修复、启动链
- 设置页面：统一使用 `Starward` 原游戏设置页

### 8.2 鸣潮

- 目标：高优先级适配
- 方向：官方接口 + 社区实现 + BSL 自建下载队列
- 目标能力：下载、更新、预下载、修复、导入、启动、卸载、缓存管理
- 设置页面：沿用米哈游同款页面结构，但后端走 BSL

### 8.3 明日方舟

- 目标：高优先级适配
- 方向：官方接口 + `Hi3Helper.Plugin.Hypergryph` 相关实现思路 + BSL 自建下载队列
- V1 结论：
  - 下载 / 更新 / 修复 / 导入 / 启动 / 卸载要做
  - 预下载当前缺少稳定可复用直连接口，V1 明确标为不可用
- 设置页面：沿用米哈游同款页面结构，但后端走 BSL

### 8.4 明日方舟：终末地

- 目标：高优先级适配
- 方向：官方接口 + 社区资料归档 + BSL 自建下载队列
- V1 结论：
  - 下载 / 更新 / 修复 / 导入 / 启动 / 卸载要做
  - 预下载当前未发现稳定可复用直连接口，V1 明确标为不可用
- 设置页面：沿用米哈游同款页面结构，但后端走 BSL

## 9. 首页与状态展示原则

- 能做的功能直接开放
- 不能做的功能必须在 UI 上写清楚
- 不伪装不可用功能为可用按钮
- 首页、任务中心、设置页尽量共用统一状态模型

## 10. 应用级要求

- 保留任务中心
- 支持单任务下载队列与排队执行
- 任务中心需要优先保证真实任务状态、进度、来源和可用操作清晰可扫
- 任务中心的 UI 设计稿仅作为设计评审材料，正式运行时仍以 WinUI 3 页面实现为准
- 后续需要提供“隐藏不想放在首页的游戏”选项
- 后续需要补应用级 BSL 设置分组

## 11. V1 明确不做

- 崩坏3 首发适配
- 其他未列出游戏的适配
- 未适配游戏的导入与本地管理
- 账号管理、自动登录、账号切换
- 抽卡、战绩、便笺等扩展能力
- 多语言
- 官方游戏多实例安装
- 全盘深度扫描已安装游戏
- 启动器自更新
