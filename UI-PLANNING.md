# BSL-Station UI Planning

> 日期：2026-06-24
> 状态：正式方案，后续只在此方案上继续收口

## 1. 唯一正式 UI 路线

- 正式 UI 主参考：`Starward`
- 正式主工程：`BSL-Station-Starward/src/Starward`
- 后续首页、任务中心、游戏设置页都继续在 `Starward` 基础上改

## 2. 不再继续的旧路线

- `XianYuLauncher` 主壳方案
- 从空白 WinUI 页面重做另一套首页
- 并行维护多套 UI 框架

## 3. 首页保留原则

- 顶部固定游戏图标区继续按 `Starward` 逻辑保留
- 游戏卡片切换逻辑继续贴近 `Starward`
- 右侧操作区继续沿用 `Starward` 启动器结构
- 米哈游游戏信息区尽量直接复用 `Starward` 现成内容链
- 非米哈游游戏在同一页面结构下接入自己的背景、公告、状态与按钮逻辑

## 4. 后续 UI 只做三类改动

1. 给真实后端状态留展示位
2. 修正和 `Starward` 仍不一致的局部细节
3. 把第三方游戏接入与米哈游一致的视觉和交互体系

HTML 设计稿只作为局部 UI 推敲材料，不作为新的运行时路线；正式实现仍以 WinUI 3 和 `Starward` 工程为准。

## 5. 页面级规划

### 首页

- 保留 `Starward` 结构作为主框架
- 米哈游游戏优先直接复用原控件与原逻辑
- 第三方游戏逐个接到相同布局下
- 不再保留给开发看的解释性文案

### 任务中心

- 保留任务中心，不删除
- 视觉方向改成更接近下载中心，而不是说明页
- 当前任务中心采用紧凑列表方向：当前任务、等待队列和历史任务分区显示
- 操作入口优先使用图标按钮和 tooltip，减少长文字按钮挤占任务信息空间
- 明确区分：
  - `Starward 原生任务`
  - `BSL 自建队列任务`
- 控制按钮只按真实后端能力显示
- 任务中心 HTML 设计稿路径：
  - `design/task-page/task-page-a-compact-list.html`
  - `design/task-page/task-page-b-split-operations.html`
  - `design/task-page/task-page-c-queue-board.html`

### 游戏设置

- 米哈游游戏统一使用 `Starward` 原游戏设置页
- 第三方游戏统一使用与米哈游同款结构的 BSL 设置页
- 第三方设置页需要优先承接：
  - 导入已安装目录
  - 修改安装目录
  - 自定义启动程序
  - 启动参数
  - 修复
  - 卸载
  - 缓存管理
  - 游戏包体 / 资源清单

### 应用设置

- 暂时继续保留 `Starward` 应用设置页
- 后续补 BSL 专属分组，不急于现在另起页面

## 6. 当前执行结论

- 当前首页方案已经足够作为正式 UI 继续推进
- 后续优先级不再是“换 UI”
- 后续优先级是“按真实后端能力逐页收口”
- 任务中心已选用“紧凑列表”方向进入 WinUI 实现，另外两个设计稿暂作为后续比较参考
